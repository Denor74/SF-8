8.6. Алгоритм бинарного поиска

Алгоритм бинарного поиска

Алгоритм поиска следующий:

  1. Определение значения элемента в середине структуры данных. Полученное значение сравнивается с ключом.
  2. Если ключ меньше значения середины, то поиск осуществляется в первой половине элементов, иначе — во второй.
  3. Поиск сводится к тому, что вновь определяется значение серединного элемента в выбранной половине и сравнивается с ключом.
  4. Процесс продолжается до тех пор, пока не будет найден элемент со значением ключа или не станет пустым интервал для поиска.

Бинарный поиск показывает хорошие результаты по скорости на больших упорядоченных списках. На неупорядоченных — бинарный поиск не работает (не имеет смысла шаг 2).

В зависимости от размера массива легко предугадать максимальное количество шагов поиска: 1 + log₂N , где N — число элементов. Разумеется, число требуется округлить до целого
